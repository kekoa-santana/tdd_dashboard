# Model Performance Dashboard Page — Implementation Spec

## Location
`tdd-dashboard/pages/model_performance.py`

## Data Sources (from `data/dashboard/`)
All parquets are generated by `precompute_dashboard_data.py` in the player_profiles repo.

| File | Contents | Generated by |
|------|----------|-------------|
| `backtest_game_prop_summary.parquet` | Fold-level metrics for all 9 props | `run_game_prop_backtest.py` → precompute |
| `backtest_confidence_tiers.parquet` | Tier assignments + explanations | precompute (confidence_tiers.py) |
| `game_prop_predictions_*.parquet` | Game-level predictions per prop | `run_game_prop_backtest.py` |
| `backtest_hitter_k_backtest.parquet` | Season-level hitter K backtest | Converted from CSV |
| `backtest_pitcher_k_backtest.parquet` | Season-level pitcher K backtest | Converted from CSV |
| `backtest_*_multi_stat_backtest.parquet` | Multi-stat season backtests | Converted from CSV |
| `backtest_*_counting_backtest.parquet` | Counting stat backtests | Converted from CSV |

## Page Layout

### Summary Scorecard (always visible)
```
┌─────────────────────────────────────────────────────┐
│  MODEL PERFORMANCE                                   │
│                                                       │
│  Overall Calibration (avg ECE): 0.042                │
│  Props beating baseline: 7 / 9                       │
│  Best edge: Pitcher K at 70%+ conf → 76% win rate   │
└─────────────────────────────────────────────────────┘
```

Load from `backtest_confidence_tiers.parquet`. Count tiers, find best performer.

### Pitcher Props Section

#### High Confidence (always expanded)
For each HIGH tier prop:
- Stat name + icon
- Key metrics: Brier, ECE, win rate at 70%+ confidence
- One-line explanation from tier data

#### Medium Confidence (collapsed by default, st.expander)
Same layout but with "why medium" explanation

#### Lower Confidence (collapsed by default)
Same layout but with "why lower" explanation

### Batter Props Section
Same tiered layout as pitcher.

### Season Projections Section
Two tables:
1. **Rate Stats** — K%, BB% for hitter/pitcher vs Marcel (from multi_stat_backtest parquets)
2. **Counting Stats** — K, BB, HR, SB, Outs vs Marcel (from counting_backtest parquets)

Load from `backtest_*_backtest.parquet` files. Show MAE, RMSE, Brier improvement.

### Deep Dives (per stat, st.expander)
For each prop, expandable section with:

1. **Calibration curve** — P(over) vs actual frequency
   - Load from prediction parquet, bin predictions, compute actual frequency
   - Plot: x=predicted prob, y=actual freq, diagonal=perfect calibration
   - Use tdd_theme colors (GOLD line, SLATE diagonal)

2. **Brier by prop line** — bar chart showing Brier score per line
   - From summary parquet, group by line

3. **Coverage chart** — 50/80/90/95% CI coverage
   - Bar chart: expected vs actual coverage
   - From summary parquet

4. **Temperature diagnostic**
   - Single number: T > 1 = overconfident, T < 1 = underconfident
   - Color: green if 0.8 < T < 1.2, yellow if 0.5-0.8 or 1.2-1.5, red otherwise

5. **Fold-over-fold stability**
   - Small table showing metrics per fold (test_season as rows)

## Graceful Degradation
- If `backtest_confidence_tiers.parquet` missing → show "Backtests pending — run `python scripts/run_game_prop_backtest.py` to generate"
- If individual prediction parquets missing → hide deep dive for that prop, show "Deep dive pending" badge
- If season backtest parquets missing → hide Season Projections section
